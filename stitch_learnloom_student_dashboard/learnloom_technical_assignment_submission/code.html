Build a Concurrent Web Scraper

Implement a high-performance web scraper in Go utilizing goroutines and channels to process multiple target domains simultaneously, adhering to polite scraping practices.

1. Objective

Develop a command-line application that reads a list of URLs from an input file, concurrently fetches their HTML content, extracts all internal and external hyperlinks, and outputs a structured JSON report detailing the link topology per domain.

2. Technical Requirements

  • Use Go 1.21+. No external scraping libraries (like Colly) are permitted for the core extraction logic; use the standard net/http and golang.org/x/net/html packages.
  • Implement a worker pool pattern. The number of concurrent workers should be configurable via a command-line flag (--workers, default: 5).
  • Respect robots.txt rules for every domain accessed.
  • Implement exponential backoff for failed requests (500, 502, 503, 504).
  • Ensure graceful shutdown upon receiving SIGINT (Ctrl+C), ensuring all currently processing URLs complete and data is flushed to disk before exiting.

3. Input Format

The application should accept a plain text file containing one URL per line:

https://example.com
https://golang.org
https://news.ycombinator.com

4. Output Format

Generate a results.json file structured as follows:

{
  "scrape_duration_ms": 1450,
  "total_urls_processed": 3,
  "results": {
    "https://example.com": {
        "status_code": 200,
        "internal_links": 4,
        "external_links": 2,
        "error": null
    }
  }
}

5. Grading Rubric

  • Concurrency Implementation (40%): Correct use of channels, WaitGroups, and worker pool. No race conditions.
  • Robustness & Error Handling (30%): Proper handling of network timeouts, invalid HTML, and server errors.
  • Code Quality (20%): Clean, modular design, idiomatic Go naming conventions, and sufficient comments.
  • Testing (10%): Basic unit tests for the HTML parsing logic.
