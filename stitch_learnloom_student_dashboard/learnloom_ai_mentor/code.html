Loomie 10:46 AM
Exactly. When you pass an inline arrow function like onClick={() => handleSelect(id)} to a child component, it creates a new function reference on every single render. If that child is wrapped in React.memo, it will still re-render because the prop reference changed.
Here is how you can stabilize that function reference using useCallback:
jsx 
import { useState, useCallback, memo } from 'react';

// 1. Wrap the child in React.memo
const ListItem = memo(({ item, onSelect }) => {
  console.log(`Rendering item ${item.id}`);
  return (
    <button onClick={() => onSelect(item.id)}>
      {item.name}
    </button>
  );
});

export default function LargeList({ items }) {
  const [selectedId, setSelectedId] = useState(null);

  // 2. Wrap the handler in useCallback
  const handleSelect = useCallback((id) => {
    setSelectedId(id);
  }, []); // Empty deps if it doesn't depend on external state

  return (
    <ul>
      {items.map(item => (
        <ListItem 
          key={item.id} 
          item={item} 
          onSelect={handleSelect} <!-- Stable reference -->
        />
      ))}
    </ul>
  );
}
Notice that ListItem is also wrapped in memo. useCallback only prevents the function reference from changing; you still need memo on the child to actually skip the render.
