### Phase 3 — Google Authentication ✅ Fixed
**Root cause:** `signInWithGoogle()` was calling `supabase.auth.signInWithSSO({ domain: 'miaoda-gg.com' })` — a proprietary enterprise SAML endpoint from a different platform. This **completely prevented** Google Sign In from working.

**Fix applied:** Replaced with standard `supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: '/auth/callback' } })`.

**Remaining setup required by operator:**
1. Create OAuth 2.0 credentials in Google Cloud Console
2. Add redirect URI: `https://<project>.supabase.co/auth/v1/callback`
3. Enable Google provider in Supabase Dashboard → Authentication → Providers
